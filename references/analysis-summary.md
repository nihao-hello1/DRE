## 五、编号体系

大学方案使用 4 套独立编号定义（numId=7~11）：

```
numId=7: 中文编号（一、二、三...）
numId=8: 正文段落编号（1. 2. 3. ...）
numId=9: 正文段落编号（技术参数列表）
numId=10: 步骤编号（1. 2. 3. ...）
numId=11: 物料清单编号
```

**观察**：标题没有使用内置 heading 样式，而是用 Normal 样式 + 手动加粗 + 编号来实现。这说明很多中文文档的标题是通过"格式刷"手动做到的，而非样式。

## 八、关键结论

1. **页边距**：2.54/2.54/3.17/3.17cm 是技术文档的通用选择
2. **正文**：宋体 12pt、1.5倍行距、首行缩进2字符 — 四份文档一致
3. **标题字体**：黑体 SimHei（智慧校园）比宋体（大学方案）更适合技术方案
4. **标题字号**：H1=16-18pt, H2=14-16pt, H3=12-15pt
5. **编号**：中文文档普遍使用 Word 多级列表编号，最常用阿拉伯数字（1. / 1.1 / 1.1.1）
6. **页眉页脚**：大约一半文档有，由具体文档类型决定 — DRE 不应默认添加
7. **表格**：蓝色系表头（#31849B或#4472C4）+ 白字是最常见的"正式"风格
